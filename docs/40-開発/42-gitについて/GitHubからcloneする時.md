# Gitからクローンを作成する手順

## 1. GitHubアカウントの確認

本リポジトリは**プライベートリポジトリ**のため、GitHubへのアクセス権限が必要です。

- GitHubアカウントがリポジトリに追加されているか、**関係者に確認してください**
- アカウントが追加されていない場合は、関係者にGitHubアカウント名を共有し、アクセス権限の付与を依頼してください

---

## 2. SourceTreeでリポジトリをクローン

### 2-1. SourceTreeを起動

- SourceTreeを起動します

### 2-2. リポジトリをクローン

1. SourceTreeのメイン画面で「**新規/クローンを作成**」をクリック
2. 「**URLからクローン**」タブを選択
3. 以下の情報を入力：
   - **ソースURL/パス**: `https://github.com/mshrynzw/aurawork.git`
   - **保存先パス**: クローン先のディレクトリを選択（例：`C:\Users\<ユーザー名>\Documents\aurawork`）
4. 「**クローン**」をクリック
5. 初回クローン時は認証が求められる場合があります：
   - **HTTPS**を使用する場合、GitHubのユーザー名とパスワード（またはPersonal Access Token）を入力
   - 認証情報はWindows資格情報マネージャーに保存されます

### 2-3. クローン完了の確認

- クローンが正常に完了すると、SourceTreeにリポジトリが表示されます
- デフォルトで`master`ブランチがチェックアウトされていることを確認してください

---

## 3. 開発用ブランチの作成

### 3-1. ブランチ名のルール

開発用ブランチは以下の形式で作成してください：

```
dev/<名前>/YYMM/DD-00
```

**例：**
- 米沢正寛の場合：`dev/m.yonezawa/2501/01-00`
- 2025年01月01日に作成する場合：`dev/<名前>/2501/01-00`

**説明：**
- `dev/`: プレフィックス（開発ブランチであることを示す）
- `<名前>`: 開発者の識別子（例：`m.yonezawa`）
- `YYMM`: 年（下2桁）と月（2桁）
- `DD`: 日（2桁）
- `-00`: 連番（同日に複数ブランチを作成する場合に使用）

### 3-2. ブランチの作成手順（SourceTree）

1. SourceTreeでリポジトリを開いた状態で、ブランチ一覧から「**master**」ブランチを選択
2. 右クリック → 「**masterから新規ブランチを作成**」を選択
   - または、画面上部の「**ブランチ**」ボタンをクリック → 「**新しいブランチ**」を選択
3. ブランチ名を入力（例：`dev/m.yonezawa/2501/01-00`）
4. 「**ブランチを作成**」をクリック
5. 作成したブランチが自動的にチェックアウトされます

### 3-3. ブランチの作成手順（コマンドライン）

SourceTreeを使用しない場合、コマンドラインでも作成できます：

```bash
# 最新のmasterブランチに移動
git checkout master
git pull origin master

# 開発ブランチを作成して切り替え
git checkout -b dev/<名前>/YYMM/DD-00

# 例：米沢正寛さん、2024年12月25日の場合
git checkout -b dev/m.yonezawa/241225-00
```

---

## 4. 確認事項

クローンとブランチ作成が完了したら、以下を確認してください：

- [ ] リポジトリが正常にクローンされている
- [ ] 開発用ブランチが作成されている
- [ ] 開発用ブランチにチェックアウトされている（SourceTreeのブランチ表示で確認）
